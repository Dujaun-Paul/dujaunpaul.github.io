Learnings

HyperText Markup Language is a computer language that annotates raw text or data on a webpage. It defines the structure and content of websites and serves as the foundation for all webpages on the World Wide Web (Web). Web browsers interpret the HTML and render it through various built-in layout mechanisms. What makes HTML hyper is its ability to link to other HTML documents, via hyperlinks, like this, expanding beyond the traditional constraints of the written word. This is what allows us to browse or surf the Web and is more efficient than typing whole URLs (Uniform Resource Locators) into search bars each time we want to locate a webpage or resource on the Web.
The language was invented in 1993, just four years after the invention of the Web, by the same inventor, Tim Berners-Lee, during his tenure at the European Organization for Nuclear Research (CERN), to facilitate the sharing of information and documents among scientists, allowing them to create a system of interconnected documents using hypertext links.
Thanks to these inventions, we're able to share and watch cat videos online and harass our friends and families with brain-rot Instagram and TikTok reels. We're currently on version five, and its standards are actively developed and maintained by the W3C (World Wide Web Consortium), which was also founded by Tim Berners-Lee.

HTML Anatomy

HTML elements, or simply elements, are the building blocks of an HTML document. They consist of opening and closing HTML tags that enclose content, and this content, whether text, images, or other elements, is an integral part of the element itself. These tags provide the structure and markup for the content, instructing web browsers on how to render the data, which ultimately defines the content and layout of a webpage.

HTML Structure

Furthermore, HTML is structured and organised as a collection of family tree relationships. This concept is known as the HTML hierarchy: the relationship between elements and their ancestor and descendant elements. Elements that are nested within other elements are called child elements or descendants, while elements that nest or contain other elements are referred to as parent elements or ancestors. Elements that are at the same level of indentation within the same parent are called siblings. The family tree metaphor is a helpful way to visualise this structure, and it's essential to remember, especially when applying CSS, as these relationships impact styling and layout.
